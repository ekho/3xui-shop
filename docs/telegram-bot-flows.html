Telegram-бот для 3x-ui — диаграммы потоков

Стек: aiogram 3 + py3xui + PostgreSQL. Оплата: крипта / Telegram Stars / карта вручную. Все потоки — MVP.

Ромб — решение/ветвление · прямоугольник — действие/состояние · «3x-ui» — вызовы панели через py3xui.

1. Регистрация и подтверждение админом

Открытая регистрация, но доступ и покупки — только после апрува.

flowchart TD
    S(["/start"]) --> N{"Уже есть аккаунт?"}
    N -->|"Нет"| C["Создать пользователя
approval_status = pending"]
    C --> NA["Уведомить админа:
карточка кандидата"]
    NA --> AD{"Решение админа"}
    AD -->|"Подтвердить"| AP["approved:
меню + доступ к триалу"]
    AD -->|"Отклонить"| RJ["rejected:
уведомление"]
    N -->|"Да"| ST{"Статус?"}
    ST -->|"pending"| W["«Ожидайте подтверждения»"]
    ST -->|"approved"| M["Главное меню"]
    ST -->|"blocked"| B["Доступ закрыт"]
    AP --> M

2. Оплата → активация (три рельса)

Заказ → выбор способа оплаты → выдача доступа в 3x-ui. Единый fulfillment для всех рельсов.

    P["Выбор тарифа

3. Продление подписки

Авто-рекуррент, если первая оплата была в Stars; иначе — напоминания по сроку и трафику.

flowchart TD
    EX["Подписка близка к концу:
по сроку ИЛИ трафику"] --> Q{"Первая оплата в Stars
и рекуррент включён?"}
    Q -->|"Да"| A1["Авто-списание Stars"]
    A1 --> A2{"Списание прошло?"}
    A2 -->|"Да"| EXT["Продлить:
extend срок + сброс трафика"]
    A2 -->|"Нет"| RM["Напоминания +
переход в ручное продление"]
    Q -->|"Нет"| RM2["Напоминания:
срок 3д / 1д / день X
трафик 80% / 100%"]
    RM2 --> U{"Пользователь продлил?"}
    RM --> U
    U -->|"Да"| PAY["Оплата: крипта / Stars / карта"]
    PAY --> EXT
    U -->|"Нет"| DIS["Grace → disable в 3x-ui"]

4. Жизненный цикл подписки (состояния)

Единая модель состояний, на которую завязаны крон-задачи и напоминания.

stateDiagram-v2
    [*] --> pending: оплата/триал подтверждены
    pending --> active: клиент создан в 3x-ui
    active --> expiring_soon: близок срок ИЛИ трафик
    expiring_soon --> active: продлено
    expiring_soon --> expired: срок/трафик исчерпан
    expired --> active: оплата продления
    expired --> disabled: после grace
    disabled --> active: реактивация при оплате
    disabled --> deleted: через N дней
    deleted --> [*]
