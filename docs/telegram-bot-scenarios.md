# Telegram-бот для 3x-ui — карта сценариев

_Черновик функциональной спецификации. Дата: 3 июля 2026._

## Зафиксированные решения

- **Без n8n.** Ядро — код: `aiogram 3` + `py3xui` + PostgreSQL + Redis + APScheduler. База — форк `snoups/3xui-shop`.
- **Панель:** 3x-ui.
- **Оплата:** крипта (CryptoBot/Cryptomus) + Telegram Stars + карта→карта вручную (админ подтверждает).
- **MVP:** пользователи — доверенные, полный антифрод не нужен. Приём оплаты — неофициальный (по налогам/легальности решение принято отдельно).

## Согласованные параметры MVP

- **Доступ:** открытая регистрация, но использование/покупка — **только после подтверждения админом** (гейт апрува). Появляется статус пользователя «ожидает подтверждения».
- **Единица подписки:** **срок + лимит трафика + лимит устройств** (все три измерения в тарифе).
- **Продление:**
  - если **первая оплата была в Stars** → предлагается/включается **авто-рекуррент через Telegram Stars** (родная подписка звёздами);
  - во всех остальных случаях (крипта, ручная карта) → **заблаговременные напоминания** об исчерпании **срока и/или трафика** с кнопкой «Продлить».
- **Триал:** входит в **MVP** (бесплатный пробный доступ после апрува).

## Акторы

- **Гость** — впервые открыл бота, ещё нет активной подписки.
- **Кандидат** — зарегистрировался, ожидает подтверждения админом.
- **Клиент** — подтверждён; есть или была подписка.
- **Реферер** — клиент, приглашающий других (Later).
- **Админ/оператор** — апрув регистраций, управление, подтверждение ручных платежей.
- **Система** — фоновые задачи (крон, вебхуки, синхронизация с панелью).

## Модель данных (черновик)

- **user**: `tg_id`, username, язык, дата регистрации, `approval_status` (pending/approved/rejected), статус (active/blocked), `referrer_id`, источник.
- **plan**: длительность (дни), лимит устройств, **лимит трафика (ГБ)**, цены по валютам (RUB/USDT/XTR), признак «доступен для триала».
- **server** (Later, мультисервер): имя, локация, endpoint панели, inbound, ёмкость.
- **order**: user, plan, server, сумма+валюта, способ оплаты, статус, промокод, `created_at`, `expires_at`.
- **payment**: order, провайдер, внешний id, статус, сумма, время, идемпотентный ключ, `is_recurring`, `telegram_payment_charge_id`.
- **subscription**: user, server, `xui_client_uuid/email`, `subscription_url`, `expires_at`, **лимит трафика, лимит устройств, использовано трафика**, флаг авто-рекуррента, статус.

## Стейт-машина пользователя (доступ)

`guest → pending_approval → approved` · ветка `pending_approval → rejected` · `approved → blocked` (бан).
Покупка/триал доступны только в состоянии `approved`.

## Стейт-машина заказа (order)

`created → awaiting_payment → paid → fulfilled`
ветки: `awaiting_payment → expired` (инвойс протух) / `→ cancelled`; `paid → failed_fulfillment` (оплата прошла, панель не создала клиента → ретрай/алерт админу).

## Стейт-машина подписки (subscription)

`none → pending → active → expiring_soon → expired → disabled → deleted`
- `pending` — оплата/триал подтверждены, клиент создаётся в 3x-ui.
- `active` — доступ выдан.
- `expiring_soon` — приближается срок **или** порог трафика (триггер напоминаний).
- `expired` — срок/трафик исчерпан; короткий grace-период.
- `disabled` — отключён в панели после grace.
- `deleted` — удалён из панели через N дней.
- Реактивация: `expired/disabled → active` при оплате продления.

# Сценарии

Легенда: **[MVP]** — в первой версии · **[Later]** — последующие итерации.

## A. Регистрация, апрув и онбординг

1. **[MVP] Первый `/start`.** Автосоздание пользователя (`approval_status=pending`). Сообщение «заявка отправлена, ожидайте подтверждения». Меню ограничено.
2. **[MVP] Апрув админом.** Админу — карточка кандидата (id, username) с кнопками **Подтвердить / Отклонить**. Подтверждён → пользователю пуш «доступ открыт» + меню/триал. Отклонён → уведомление.
3. **[MVP] Повторный `/start`.** Узнаём пользователя → показываем статус (ожидает/активен) и меню.
4. **[MVP] Триал после апрува.** «Попробовать бесплатно» → создание клиента в 3x-ui на N дней с триал-лимитами (трафик/устройства). Один раз на пользователя.
5. **[MVP] Выбор/автоопределение языка.** Ru/En.
6. **[Later] Оферта/условия.** Показ и подтверждение при первом входе.
7. **[Later] Реферальный вход.** `/start ref_<id>` → привязка реферера.

## B. Выбор тарифа и оформление заказа

8. **[MVP] Каталог тарифов.** Срок + лимит трафика + лимит устройств + цена (по валютам).
9. **[Later] Выбор локации/сервера.** Если несколько 3x-ui — выбор страны; распределение по пулу.
10. **[MVP] Оформление заказа.** Создаётся `order` в статусе `awaiting_payment`.
11. **[MVP] Выбор способа оплаты:** крипта / Stars / карта вручную.
12. **[Later] Промокод.** Ввод кода до оплаты → пересчёт суммы/доп. дни.

## C. Оплата — крипта (CryptoBot / Cryptomus)

13. **[MVP] Выставление инвойса.** `createInvoice` → кнопка/ссылка оплаты.
14. **[MVP] Подтверждение оплаты.** Вебхук `paid` → активация (см. F) → выдача доступа. Продление — по напоминаниям (см. G).
15. **[MVP] Инвойс не оплачен/протух.** По таймауту `order → expired`, предложение оформить заново.
16. **[MVP] Идемпотентность вебхука.** Повторный колбэк не создаёт вторую подписку.
17. **[Later] Недоплата/переплата.** Underpaid → `awaiting`; overpaid → зачёт/уведомление админу.

## D. Оплата — Telegram Stars

18. **[MVP] Инвойс Stars.** `sendInvoice` (XTR) → `pre_checkout_query` (ответить ≤10 c) → `successful_payment` → активация.
19. **[MVP] Отклонение pre-checkout.** Товар недоступен/ошибка → корректный отказ.
20. **[MVP] Авто-рекуррент Stars.** Если первая оплата звёздами — оформить/предложить родную Stars-подписку с автосписанием; управление и отмена из «Моей подписки».
21. **[Later] Возврат Stars (refund).** По обращению/ошибке — по `telegram_payment_charge_id`.

## E. Оплата — карта→карта (ручная)

22. **[MVP] Инструкция и реквизиты.** Карта/реквизиты + сумма + комментарий + шаги.
23. **[MVP] «Я оплатил».** Пользователь подтверждает (опц. прикладывает скрин/чек).
24. **[MVP] Модерация админом.** Админу — карточка заказа с кнопками **Подтвердить / Отклонить** (+ вложение).
25. **[MVP] Решение админа.** Подтвердил → активация; отклонил → уведомление с причиной.
26. **[Later] Уникальная сумма-нонс + авто-сопоставление** по входящим уведомлениям банка + дедуп.
27. **[Later] OCR чека** как подсказка оператору (не основание для выдачи).

## F. Выдача доступа и подписка (клиент)

28. **[MVP] Активация в 3x-ui.** Создать клиента (uuid/email, expiry, **лимит трафика, лимит устройств**) через `py3xui`, сгенерировать subscription-URL / VLESS-ссылку / QR.
29. **[MVP] Инструкция по подключению.** Гайд по ОС (iOS/Android/Windows/macOS) и клиентам (v2rayTun/Hiddify и т.п.), deeplink-импорт, QR.
30. **[MVP] «Моя подписка».** Статус, срок до, **остаток трафика**, устройства, ссылка/QR, кнопка «Продлить», управление авто-рекуррентом Stars (если включён).
31. **[MVP] Продление.** Повторная оплата → **extend** срока/сброс трафика у существующего клиента (не пересоздавать).
32. **[Later] Смена/сброс ссылки** при компрометации (перегенерация uuid/subId).
33. **[Later] Апгрейд/смена тарифа** (устройства/трафик/локация), пересчёт.

## G. Жизненный цикл (система, крон)

34. **[MVP] Напоминания по сроку.** За 3 дня / 1 день / в день X — с кнопкой «Продлить».
35. **[MVP] Напоминания по трафику.** При 80% / 100% лимита — с предложением продлить/сменить тариф.
36. **[MVP] Авто-отключение.** По исчерпании срока **или** трафика (после короткого grace) → `disable` в панели.
37. **[MVP] Идемпотентные фоновые проходы.** Без дублей уведомлений и действий.
38. **[Later] Удаление после N дней.** Чистка отключённых клиентов из панели.
39. **[Later] Синхронизация бот↔панель.** Подтягивать ручные изменения из 3x-ui.

## H. Реферальная программа [Later]

40. Реф-ссылка. 41. Бонус рефереру при оплате приглашённого. 42. Реф-статистика.

## I. Промокоды и баланс [Later]

43. Промокод (скидка/доп. дни). 44. Внутренний баланс: пополнение и оплата с баланса.

## J. Поддержка и сервис

45. **[MVP] «Поддержка».** Связь с админом (пересылка / прямой контакт).
46. **[MVP] Уведомление о ТО.** Maintenance mode.
47. **[Later] FAQ / база инструкций.**

## K. Админские сценарии

48. **[MVP] Апрув/отклонение регистраций** (см. A2).
49. **[MVP] Поиск/просмотр пользователя** — подписки, заказы, платежи, трафик.
50. **[MVP] Ручное управление** — выдать/продлить/отключить/забанить, начислить дни/трафик.
51. **[MVP] Модерация ручных платежей** (см. E).
52. **[MVP] Рассылка** — всем/сегменту, с предпросмотром.
53. **[MVP] Бэкап БД** и **Maintenance mode**.
54. **[MVP] Настройка триала** — длительность и лимиты.
55. **[Later] Управление серверами/пулом** (мультисервер).
56. **[Later] Тарифы / промокоды** — CRUD из бота.
57. **[Later] Аналитика** — доход, активные, конверсия, отток.

## L. Крайние случаи и ошибки

58. **[MVP] Оплата прошла, панель упала.** `paid`, но клиент не создан → ретраи + алерт админу + понятный статус (без потери оплаты).
59. **[MVP] Панель недоступна.** Очередь/ретрай, дружелюбное сообщение.
60. **[MVP] Дубликат платежа/вебхука.** Идемпотентность по внешнему id.
61. **[MVP] Пользователь заблокировал бота.** Уведомления помечаются недоставленными.
62. **[MVP] Сбой авто-рекуррента Stars.** Списание не прошло → напоминание + перевод в ручное продление.
63. **[Later] Двойной/параллельный заказ.** Блокировка повторной оплаты.
64. **[Later] Возвраты/спор.** Крипта — нет; Stars — refund; карта — вручную.
65. **[Later] Переезд подписки** на другой tg-аккаунт.

## MVP-срез (первая версия)

1. **A1–A5** — регистрация → апрув админом → онбординг → **триал**, ru/en.
2. **B8, B10, B11** — каталог (срок+трафик+устройства) → заказ → выбор оплаты.
3. **C13–C16 / D18–D20 / E22–E25** — крипта, Stars (+авто-рекуррент), ручная карта с модерацией.
4. **F28–F31** — создание клиента в 3x-ui с лимитами, выдача конфига/инструкции, «моя подписка», продление.
5. **G34–G37** — напоминания по сроку и трафику, авто-отключение.
6. **J45–J46** — поддержка, maintenance.
7. **K48–K54** — админка: апрув, управление, модерация платежей, рассылка, бэкап, настройка триала.
8. **L58–L62** — базовая устойчивость (без полноценного антифрода).

Во вторую очередь: H/I (рефералы, промо, баланс), мультисервер, авто-матчинг платежей и OCR, аналитика, апгрейды тарифа.

## Осталось решить

- **Мультисервер:** одна нода в MVP (заложено) или сразу пул с выбором локации.
- **Grace-период** после истечения: сколько дней доступ ещё жив до `disable`.
- **Триал-лимиты:** длительность, трафик, число устройств.
- **Валюты отображения** и источник курса (RUB/USDT/XTR).
