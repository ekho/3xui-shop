# Renewal Site — One-Pager (v1 MVP)

_Мостик от брейншторма к постройке. Без кода — но с границами и якорями. Дата: 2026-07-07._

---

## 1. Проблема

Продукт — Telegram-бот продажи VPN. Telegram в РФ душат/блокируют. Юзер с **истёкшей** подпиской попадает в тупик: VPN не работает → не может через него зайти в придушенный Telegram → не может продлиться в боте → остаётся оффлайн. Ровно тот, кому нужно продлить, отрезан от единственного канала продления.

## 2. JTBD

> «Когда моя подписка истекла и Telegram недоступен, я хочу продлить её и вернуться онлайн за минуту — **без Telegram и без работающего VPN**.»

## 3. Ключевые решения (зафиксированы)

- **Скоуп v1 — только продление для уже купивших**, кто лапснулся. Не привлечение новых, не витрина, не выдача конфигов.
- **Опознание без Telegram** — по `sub_id` (хвост subscription-ссылки, которую юзер уже держит). Не аккаунты, не пароли.
- **Оплата** — существующими web-шлюзами (Cryptomus/Heleket/YooKassa/YooMoney), которые уже отдают checkout-URL и ловят колбэки вебхуками. Telegram Stars — не для сайта.
- **Конфиг не переотдаём** — `extend` двигает только `expiryTime`, `sub_id`/ссылка не меняются → «продлил → переподключись».
- **Доставка renewal-URL юзеру** — через пред-экспайр напоминание в боте (пока Telegram/VPN ещё работают); сайт = insurance для хвоста.
- **Архитектура — вариант A** (панель отдаёт подписку напрямую), но `server.subscription_url` держим переключаемым, чтобы вариант B (бот как edge подписки) стал миграцией, а не переделкой.

## 4. MVP-скоуп — флоу (одна страница)

```
[юзер вставляет свою sub-ссылку или UUID]
        │
        ▼
  опознать → найти подписку по sub_id (или vpn_id)
        │
        ▼
  показать: тариф, когда истекла, цена продления
        │
        ▼
  оплатить → web-checkout (крипта / карта-СБП)
        │
        ▼  (webhook провайдера, без Telegram)
  extend_subscription() → expiryTime += duration в панели
        │
        ▼
  СТРАНИЦА сама показывает «готово, переподключись»   ← НЕ через Telegram
```

Подтверждение показывает **сама страница** (синхронно или поллит статус платежа). `NotificationService` (Telegram) в веб-путь **не тащим** — смысл фолбэка в том, что Telegram может лежать.

## 6. Риски и тесты ДО кода

1. **[Блокер] Достижимость домена сайта** лапснутым юзером без VPN при придушенном TG. → Тест: открыть будущий домен из РФ-сети без VPN под троттлингом. Митигация: Cloudflare, крипто-оплата, ранний посев URL через напоминание.
2. **[Гейт A и B] Что панель v3.4.2 отдаёт на sub-URL истёкшего клиента** — контент с заголовками, пусто или 404? Если пусто — бэкап-канал через ответ подписки мёртв, вся ставка на напоминание. → Тест на живой панели (нельзя узнать из кода; ср. `memory/b6-live-validation.md`).
3. **Bearer-семантика `sub_id`** — владение ссылкой = право продлить. Для «продли свою подписку» ок (в худшем случае чужой оплатит тебе продление). Но конфиг вшивает `email = tg_id` → ссылка палит Telegram-id (это pre-existing, не вносим нового). → Решить: достаточно ли possession, или нужен 2-й фактор.
4. **Censorship-ранжирование платежей** — крипта (Cryptomus/Heleket) устойчивее к RKN, но их checkout-домен тоже блокируем; YooKassa/СБП домашние = достижимее внутри РФ, но с KYC. → Скан не закончен (агент упал на лимите), доделать перед выбором дефолтного метода.

## 7. Явно НЕ в v1

- ❌ Привлечение новых юзеров через сайт (онбординг, витрина).
- ❌ Выдача/показ конфигов и QR на сайте (для продления не нужно).
- ❌ Система аккаунтов, логин/пароль, e-mail.
- ❌ Полноценный Mini App внутри Telegram (публичная дверь — сначала).
- ❌ Вариант B (бот как edge подписки) — только задел под миграцию.
- ❌ Автоплатёж — отдельный, комплементарный трек (см. ниже).

## 8. Решить перед стартом

- **Дефолтный платёжный метод** для сайта (зависит от п.6.4).
- **Домен(ы)** и хостинг-стратегия сайта.
- **Автоплатёж как параллельный трек** — по ROI может бить сильнее сайта (продление до истечения снимает зависимость от TG для большинства). Сайт ловит только тех, у кого автоплатёж отвалился / кто на разовых платежах.

## 9. Как поймём, что сработало

- Доля лапснувших, кто продлился через сайт, а не потерян (retention хвоста).
- Медианное время «подписка истекла» → «снова онлайн».
- Доля напоминаний, после которых URL реально сохранён/использован вне бота.
